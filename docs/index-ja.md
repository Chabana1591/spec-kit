# Spec Kit
> Spec Kitについて

*Build high-quality software faster.*
> 高品質なソフトウェアをより速く構築しましょう。

**An effort to allow organizations to focus on product scenarios rather than writing undifferentiated code with the help of Spec-Driven Development.**
> Spec-Driven Developmentを活用し、組織が差別化されていないコードではなくプロダクトシナリオに集中できるよう支援する取り組みです。

## What is Spec-Driven Development?
> Spec-Driven Developmentとは?

Spec-Driven Development **flips the script** on traditional software development. For decades, code has been king — specifications were just scaffolding we built and discarded once the "real work" of coding began. Spec-Driven Development changes this: **specifications become executable**, directly generating working implementations rather than just guiding them.
> Spec-Driven Developmentは従来のソフトウェア開発の考え方を**ひっくり返します**。何十年もの間、コードこそが主役であり、仕様は「本当の作業」であるコーディングが始まると捨てられる足場にすぎませんでした。Spec-Driven Developmentはこれを変革し、**仕様が実行可能になり**、実装を導くだけでなく直接稼働する成果物を生成します。

## Core Philosophy
> 中心となる理念

Spec-Driven Development is a structured process that emphasizes:
> Spec-Driven Developmentは、次の点を重視する構造化されたプロセスです。

- **Intent-driven development** where specifications define the "_what_" before the "_how_"
> **意図駆動型開発** — 「どのように」よりも先に「何を」仕様で定義します。
- **Rich specification creation** using guardrails and organizational principles
> **充実した仕様作成** — ガードレールと組織的な原則を活用します。
- **Multi-step refinement** rather than one-shot code generation from prompts
> **多段階のブラッシュアップ** — プロンプトからの一括コード生成ではなく段階的に洗練します。
- **Heavy reliance** on advanced AI model capabilities for specification interpretation
> **高度なAIモデルの能力に大きく依存**し、仕様を解釈します。

## Development Phases
> 開発フェーズ

| Phase | Focus | Key Activities |
|-------|-------|----------------|
| **0-to-1 Development** ("Greenfield") | Generate from scratch | <ul><li>Start with high-level requirements</li><li>Generate specifications</li><li>Plan implementation steps</li><li>Build production-ready applications</li></ul> |
| **Creative Exploration** | Parallel implementations | <ul><li>Explore diverse solutions</li><li>Support multiple technology stacks & architectures</li><li>Experiment with UX patterns</li></ul> |
| **Iterative Enhancement** ("Brownfield") | Brownfield modernization | <ul><li>Add features iteratively</li><li>Modernize legacy systems</li><li>Adapt processes</li></ul> |
> 上表は開発フェーズごとのフォーカスと主な活動を示しています。

| フェーズ | フォーカス | 主な活動 |
|-----------|------------|-----------|
| **0-to-1 Development**（グリーンフィールド） | ゼロからの生成 | <ul><li>高レベルな要件から開始</li><li>仕様を生成</li><li>実装手順を計画</li><li>本番運用可能なアプリケーションを構築</li></ul> |
| **Creative Exploration** | 並行実装 | <ul><li>多様なソリューションを探求</li><li>複数の技術スタックとアーキテクチャをサポート</li><li>UXパターンを試行</li></ul> |
| **Iterative Enhancement**（ブラウンフィールド） | 既存システムの近代化 | <ul><li>機能を段階的に追加</li><li>レガシーシステムをモダナイズ</li><li>プロセスを適応</li></ul> |

## Experimental Goals
> 実験的な目標

Our research and experimentation focus on:
> 私たちの研究と実験は次の領域に焦点を当てています。

### Technology Independence
> テクノロジーからの独立性

- Create applications using diverse technology stacks
> 多様な技術スタックを用いてアプリケーションを作成します。
- Validate the hypothesis that Spec-Driven Development is a process not tied to specific technologies, programming languages, or frameworks
> Spec-Driven Developmentが特定のテクノロジー、プログラミング言語、フレームワークに依存しないプロセスであるという仮説を検証します。

### Enterprise Constraints
> エンタープライズの制約

- Demonstrate mission-critical application development
> ミッションクリティカルなアプリケーション開発を実証します。
- Incorporate organizational constraints (cloud providers, tech stacks, engineering practices)
> 組織の制約（クラウドプロバイダー、技術スタック、エンジニアリング慣行）を取り入れます。
- Support enterprise design systems and compliance requirements
> エンタープライズのデザインシステムとコンプライアンス要件を支援します。

### User-Centric Development
> ユーザー中心の開発

- Build applications for different user cohorts and preferences
> 異なるユーザー層や嗜好に合わせたアプリケーションを構築します。
- Support various development approaches (from vibe-coding to AI-native development)
> 多様な開発アプローチ（バイブコーディングからAIネイティブ開発まで）を支援します。

### Creative & Iterative Processes
> 創造的かつ反復的なプロセス

- Validate the concept of parallel implementation exploration
> 並行実装探索というコンセプトを検証します。
- Provide robust iterative feature development workflows
> 堅牢な反復的機能開発ワークフローを提供します。
- Extend processes to handle upgrades and modernization tasks
> アップグレードやモダナイゼーションにも対応できるプロセスを拡張します。
